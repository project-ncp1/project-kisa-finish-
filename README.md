KISA UNIX 취약점 점검 자동화 스크립트 (U-07, U-08, U-10, U-15, U-20) <프로젝트 개요> 본 프로젝트는 KISA 주요 UNIX 취약점 항목을 기준으로, 운영체제 환경(macOS / Linux)을 고려한 계정·권한·파일·서비스 보안 상태를 자동 점검하기 위해 제작한 Bash 기반 보안 진단 스크립트입니다.
본 프로젝트는 Rocky-9.7-x86_64-dvd 환경에서 진행하였습니다.
----------------------------------------------------------------------------------------------------------------------------------------------
<1.OS 환경 구분 및 그룹 허용 정책 분기> -점검 목적-

운영체제(macOS / Linux)에 따라 시스템 기본 그룹 정책이 다르다는 점을 반영하여 잘못된 권한 판정을 방지하기 위함

구현 이유 -
macOS는 wheel 그룹이 관리자 역할을 수행

Linux는 기본적으로 root 그룹만 관리자 그룹으로 사용

실습 환경이 macOS 노트북이었기 때문에 리눅스 기준 스크립트를 그대로 사용할 수 없었음

보안적 의미 -
OS 환경을 고려하지 않으면 정상 시스템을 오탐할 수 있음
----------------------------------------------------------------------------------------------------------------------------------------------
2.root 계정 로그인 상태 점검 (U-07)

<점검 목적> Root 계정 직접 로그인을 차단하여 계정 탈취 및 브루트포스 공격 위험을 최소화

<점검 기준 및 구현 설명>

/etc/shadow 파일에서 root 계정의 비밀번호 필드를 확인, 비밀번호가 !!로 설정되어 있는지 여부를 점검하여 !!로 설정된 경우 비밀번호 기반 인증이 잠금 상태로 root 계정의 직접 로그인이 차단된 것으로 판단.
<구현 이유>

root 계정은 최고 권한 계정이므로 공격 성공 시 피해가 크기 때문.

password 기반 로그인 방식은 브루트포스 공격에 취약하기 때문.

<보안적 의미>
root 계정의 직접 로그인을 차단함으로써, 외부 공격자가 시도할 수 있는 침입 경로를 줄이고, 시스템의 공격 표면을 효과적으로 축소할 수 있다.
----------------------------------------------------------------------------------------------------------------------------------------------
<3./etc/shadow 소유자 권한 확인 (U-08)>

<점검 목적>

시스템 사용자 비밀번호 해시 정보가 저장된 /etc/shadow 파일의 기밀성과 무결성 보호하기 위함입니다.
<구현 이유>

/etc/shadow 파일의 소유자 및 권한을 확인하여 인증 정보가 저장된 파일에 대한 비인가 접근 및 변조를 방지하기 위한것으로 해당 파일이 노출될 경우 해시 값 탈취 후 오프라인 크래킹 공격이 가능하며, 파일 변조 시 계정 탈취 및 권한 상승 공격으로 이어질 수 있기 때문입니다.
<보안적 의미> -파일 소유자는 root 계정으로 유지돼야 합니다. -파일 권한은 400으로 설정하여 root만 읽기 가능하도록 제한합니다. -이는 최소 권한 원칙을 준수하기 위함.
----------------------------------------------------------------------------------------------------------------------------------------------
<4. /etc/hosts 파일 소유자 및 권한 점검 (U-09)>

<점검 목적>

로컬 네임 해석 무결성 보호를 위함.
<구현 이유>

리눅스 시스템은 일반적으로 /etc/nsswitch.conf 설정에 따라 DNS조회 이전에 /etc/hosts 파일을 우선 참조한다. 해당 파일이 비인가 사용자에 의해 변조될 경우 특정 도메인이 공격자 서버로 매핑되어 리다이렉션, 피싱, 악성 업데이트 서버 유도 등의 보안 위협으로 이어질 수 있기 때문이다. /etc/hosts는 도메인 → IP 매핑에 직접 영향
<보안적 의미>

로컬 이름 해석 체계의 무결성 확보
최소 권한 원칙 적용
----------------------------------------------------------------------------------------------------------------------------------------------
<5.파일 및 디렉터리 소유자 설정 점검 (U-10)>

<점검 목적>

중요 시스템 파일의 그룹 소유자를 관리자 그룹으로 제한하여 비인가 그룹 접근을 방지하기 위함.
<구현 이유>

파일 그룹이 일반 사용자 그룹으로 설정되어 있고 그룹 쓰기 권한이 허용될 경우, 해당 파일의 설정 변조를 통해 인증 우회, 혹은 권한 상승으로 이어질 수 있기 때문입니다.
<보안적 의미>

파일 접근 통제를 강화하여 최소 권한 원칙을 준수하고, 설정 파일 무결성을 보호하는데 의미가 있습니다
----------------------------------------------------------------------------------------------------------------------------------------------
<6.World Writable 파일 점검 (U-15)>

<점검 목적>

시스템 내 중요 파일의 무결성을 보호하고, 비인가 사용자에 의한 파일 변조 가능성을 사전에 차단하기 위함입니다.
<구현 이유>

world writable 파일은 소유자,소유그룹, 그외의 유저들 모두가 해당 파일을 수정할 수 있음을 의미하며, 중요 파일 또는 설정 파일이 이러한 권한을 가질 경우 악성 코드 삽입 및 권한 상승 공격으로 이어질 수 있기 때문에 사전에 방지하고자 구현을 하였습니다.
<보안적 의미>

불필요한 world writable 권한을 제거함으로써 최소 권한 원칙을 적용하고 시스템 파일의 무결성을 유지하는 데 보안적인 의미가 있습니다.
----------------------------------------------------------------------------------------------------------------------------------------------
<7.Anonymous FTP 활성화/비활성화 점검 (U-20)>

<점검 목적>

Anonymous FTP는 인증 절차 없이 외부 사용자의 접근을 허용하는 기능으로, 불필요하게 활성화될 경우 시스템 정보 노출 및 무단 파일접근, 악성 파일 업로드 등의 보안 위협으로 이어질 수 있기에, 익명 접속 기능의 활성화 여부와 관련 계정 및 서비스 실행 상태를 점검하여 불필요한 외부 접근을 사전에 차단하기 위함입니다.
<보안적 의미>

불필요한 공개 서비스를 제거함으로써 공격 표면을 최소화하고, 인증 기반 접근 통제를 강화하는 데 의미가 있습니다.
